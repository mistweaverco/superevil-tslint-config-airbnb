# `variable-name`

Allow only `lowerCamelCased` or `UPPER_CASED` variable names.

## Rationale

Helps maintain a consistent, readable style in your codebase.

## Implementation

```json
{
  "rules": {
    "variable-name": [true, "check-format"]
  }
}
```

See: https://palantir.github.io/tslint/rules/variable-name/
